We have a problem:

Bandwidth

Nothing we make should require abnormally fast connections.

If you require a loading screen, you're doing it wrong.

Guiding principles

  1. The user should download as few things as possible.
  2. The things should be as small as possible.
  3. Different devices might need different things.

Images

Sizing

Your image should never be wider than 1500px.

Cropping

Most screens have a 16:9 aspect ratio.

Crop your images to be 16:9 so they fit on the screen.

Thus, a full size image would be 1500px x 844px.

Format

.jpg FOREVER

GIFS!

GIFs are great. They can be very effective.

The catch with GIFs

They are inefficient and tend to be very big in file size.

GIFs should be short, repetitive animations.

Video

When you can, use Vimeo.

When you can't use Vimeo

  • Music rights!
  • Advanced interactive components
  • You go over the free account limits

Autoplaying Video

Don't do it. It won't work on mobile anyway.

Audio

CAVEAT:

Web browsers suck at audio.

It is tempting to use SoundCloud.

When you can't use SoundCloud

  1. Music rights!
  2. When you don't want to use their incredibad super ugly player

Native audio

Native audio requires both mp3 files and ogg files to support all browsers.

jPlayer is an okay library for interacting with audio.

See me if you want to use it.

Your code!

Minification

Removing all the whitespace, comments and other unnecessary characters from your code.

Even whitespace takes up space in your file size.

Always minify your code in production.

Always use minified versions of libraries.

Bundling

Combining all code of the same language into one file.

Steps for effective minification and bundling

  1. Move all JavaScript into one file.
  2. Move all CSS into one file.
  3. Minify those files and use them in your HTML.
